Description

This module provides an implementation along with functions for working with tries. Tries allow for a fast and scalable implementation of autocomplete.

Trie + autocomplete examples in a REPL environment

>>> import Tries
>>> mytrie = trieFromList [(30, "an"), (12, "to"), (25, "ted"), (20, "a"), (14, "tea")]
>>> mytrie
a 20.0 30.0
  n 30.0 -1.0
t -1.0 25.0
  o 12.0 -1.0
  e -1.0 25.0
    a 14.0 -1.0
    d 25.0 -1.0
>>> mytrie_augmented = insertWords [(5, "in"), (15, "inn")] mytrie
>>> mytrie_augmented
a 20.0 30.0
   n 30.0 -1.0
t -1.0 25.0
   o 12.0 -1.0
   e -1.0 25.0
      a 14.0 -1.0
      d 25.0 -1.0
i -1.0 15.0
   n 5.0 15.0
      n 15.0 -1.0
isInTrie "inn" mytrie
False
isInTrie "inn" mytrie_augmented
True
>>> removePrefix "inn"  mytrie_augmented
a 20.0 30.0
   n 30.0 -1.0
t -1.0 25.0
   o 12.0 -1.0
   e -1.0 25.0
      a 14.0 -1.0
      d 25.0 -1.0
i -1.0 5.0
   n 5.0 -1.0
>>> updateWeight sqrt "ted" mytrie_augmented
a 20.0 30.0
   n 30.0 -1.0
t -1.0 14.0
   o 12.0 -1.0
   e -1.0 14.0
      a 14.0 -1.0
      d 5.0 -1.0
i -1.0 15.0
   n 5.0 15.0
      n 15.0 -1.0
>>> mytrie_augmented -- Notice that this trie was never modified.
a 20.0 30.0
   n 30.0 -1.0
t -1.0 25.0
   o 12.0 -1.0
   e -1.0 25.0
      a 14.0 -1.0
      d 25.0 -1.0
i -1.0 15.0
   n 5.0 15.0
      n 15.0 -1.0
>>> autocomplete "t" mytrie 3
[(25.0,"ted"),(14.0,"tea"),(12.0,"to")]
>>> autocomplete "in" mytrie 5
[]
>>> autocomplete "in" mytrie_augmented 5
[(15.0,"inn"),(5.0,"in")]

Check to see if a prefix is in a Trie.

getMaxWeight :: Trie -> Double Source #

Get the maximum weight in a Trie. This includes the weight of the current node and all children.

updateMaxChildWeight :: Trie -> Trie Source #

Update the max_child_wt of the current node (not of any children). Warning: this function assumes that all children have accurate max_child_wt values This is used to update a Trie whose max_child_wt value becomes stale once a child is deletmax_weighted (for example).

Inserting nodes into Tries

insertChild :: Trie -> Trie -> Trie Source #

Insert a child Trie into a parent trie and update the max_child_wt of the parent.

insertChildFast :: Trie -> Trie -> Trie Source #

Fast version of insertChild. This does NOT update max_child_wt for arbitrary insertions - this is only for adding new children to tries that are otherwise not updated. This will throw an error if the child is not valid (e.g. "the" is not a valid child of "t").

insertPrefix :: Double -> String -> Trie -> Trie Source #

Insert a prefix and its weight into the Trie.

insertWords :: [(Double, String)] -> Trie -> Trie Source #

Insert weighted prefixes into the Trie. This is essentially a wrapper that performs insertPrefix many times.

trieFromList :: [(Double, String)] -> Trie Source #

Build a Trie from a list of weighted words.

pruneChildren :: Double -> Trie -> Trie Source #

Run pruneTrie on children whose maximum weight (max(weight, max child weight)) is greater than or equal to threshold. This is effectively just a helper function for pruneTrie.

Autocomplete

autocompletePriority :: (Double, String) -> Double Source #

Give a priority for autocomplete. This is mainly because MinMaxQueue is a min-priority queue, so we need to flip the priority by negating the weight of the word.

mmqMinWeight :: MinMaxQueue Double (Double, String) -> Double Source #

Get the minimum weight in the minmaxqueue The smallest weight is actually the "maximum" in the MMQ since the priorities are the negative weights

mmqInsert :: Double -> String -> MinMaxQueue Double (Double, String) -> MinMaxQueue Double (Double, String) Source #

Insert the weighted word (the tuple (weight, word)) into q

getMatchingPrefixes :: String -> String -> Int -> Trie -> MinMaxQueue Double (Double, String) -> MinMaxQueue Double (Double, String) Source #

Get all matching terms for a given prefix. This does the majority of the work for autocomplete.

autocomplete :: String -> Trie -> Int -> [(Double, String)] Source #

Produce the top k words in a given Trie that match a particular prefix, sorted by weight.

Produce the top k words in a given list of weighted words that match a particular query, sorted by weight.
